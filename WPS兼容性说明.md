# WPS vs 微软Office Word文档兼容性说明

## 问题原因

### WPS和微软Office的主要区别

1. **文件格式标准**
   - **微软Office**: 严格遵循OOXML (Office Open XML) 国际标准
   - **WPS Office**: 基本遵循OOXML，但有自己的扩展和差异

2. **内部XML结构**
   - **微软Office**: 标准的XML命名空间和结构
   - **WPS Office**: 可能包含自定义命名空间和扩展标签
     - 例如: `http://schemas.wps.cn/officeDocument/2006/main`

3. **文档元素处理**
   - **微软Office**: 标准的段落、表格、样式结构
   - **WPS Office**: 可能有额外的样式信息和格式标记

4. **编码和格式**
   - **微软Office**: 标准UTF-8编码
   - **WPS Office**: UTF-8编码，但可能有BOM(字节顺序标记)

## 已实现的解决方案

### 双重解析机制

程序现在支持两种解析方式：

#### 方式1: 标准python-docx解析
- 适用于标准的微软Office文档
- 使用python-docx库的标准API
- 解析段落和表格内容

#### 方式2: XML直接解析(兼容模式)
- 当标准方式失败时自动启用
- 直接解析docx内部的XML结构
- 支持WPS自定义命名空间
- 兼容WPS特有的文档格式

### 改进内容

1. **增加了表格解析支持**
   - 如果文档没有段落，自动尝试解析表格内容

2. **增加了WPS命名空间支持**
   - 支持 `http://schemas.wps.cn/officeDocument/2006/main`
   - 兼容WPS特有的XML标签

3. **增强了错误处理**
   - 标准解析失败时自动切换到兼容模式
   - 提供详细的错误信息

## 使用建议

### 最佳实践

1. **推荐格式**: 使用微软Office创建价格表文档
2. **WPS用户**: 在WPS中保存时选择"微软Office兼容模式"
3. **文件格式**: 优先使用 `.docx` 格式，避免使用 `.doc` 格式

### WPS文档处理方法

如果使用WPS创建文档，建议：

#### 方法1: 另存为兼容格式
```
WPS中: 文件 → 另存为 → 选择"Microsoft Word文档(*.docx)"
```

#### 方法2: 导出为微软格式
```
WPS中: 文件 → 导出为 → Microsoft Word文档
```

#### 方法3: 使用微软Office重新保存
```
用微软Word打开 → 文件 → 另存为 → 选择.docx格式
```

## 常见问题

### Q1: 为什么WPS文档无法导入？

**A**: 可能的原因：
- WPS使用了非标准的XML结构
- 文档包含WPS特有的格式标记
- python-docx库无法识别WPS扩展

**解决方案**: 已实现自动兼容模式，现在应该可以正常导入

### Q2: 导入后数据不完整怎么办？

**A**: 检查以下几点：
- 文档格式是否为表格形式
- 数据是否按照标准格式排列
- 尝试在WPS中另存为微软格式

### Q3: 如何确认文档格式？

**A**: 
- 查看文件扩展名（应为.docx）
- 使用分析工具检查内部结构
- 尝试用微软Word打开验证

## 技术细节

### 支持的命名空间

```xml
<!-- 微软Office标准 -->
xmlns:w="http://schemas.openxmlformats.org/wordprocessingml/2006/main"

<!-- WPS扩展 -->
xmlns:wps="http://schemas.wps.cn/officeDocument/2006/main"
```

### 解析流程

```
开始解析
    ↓
尝试标准python-docx解析
    ↓
成功? → 是 → 提取文本 → 完成
    ↓
    否
    ↓
尝试XML直接解析
    ↓
成功? → 是 → 提取文本 → 完成
    ↓
    否
    ↓
返回空列表
```
