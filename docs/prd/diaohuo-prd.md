# 调货助手 — 产品需求文档（PRD）

**版本**：v1.06  
**日期**：2026-06-18  
**定位**：面向 IT 硬件经销商的一站式桌面管理工具

---

## 一、需求背景

IT 硬件经销商（俗称"调货商"）的日常业务涉及上游价格表解析、库存管理、客户报价、出库发货、收款对账等多个环节。目前大多数经销商依赖 Excel 表格或手工记录，存在以下痛点：

- 上游每周发送 Word 价格表，手动录入机型信息耗时且容易出错
- 库存分散在多个批次，难以实时掌握剩余数量和资金占用
- 报价依赖记忆和经验，新客户询价时缺乏历史数据参考
- 出库流程繁琐（SN 录入、校验、去重、开单），容易遗漏
- 账款管理混乱，应收应付缺乏系统跟踪
- 月底经营复盘依赖手工统计，无法及时发现滞销品和回款问题

调货助手应运而生，旨在用一套本地化桌面工具覆盖从进货到收款的完整业务闭环，并通过智能技能模块提供数据驱动的经营辅助。

---

## 二、产品定位与目标

### 2.1 产品定位

调货助手是一款面向 IT 硬件经销商（笔记本、台式机、配件等）的**一站式桌面管理工具**，采用完全本地化架构（无需联网、无需服务器），覆盖进货、库存、报价、出库、收款、对账的全流程。

### 2.2 核心目标

| 目标 | 衡量标准 |
|------|---------|
| 提升报价效率 | 从翻记录/心算 → 一键获取历史区间和建议价 |
| 降低库存风险 | 实时库存扣减 + 滞销预警 + 价格异动检测 |
| 减少出库差错 | SN 批量扫描/校验/去重 + 出库确认单 |
| 清晰账款管理 | 应收应付一目了然，FIFO 自动分配收款 |
| 辅助经营决策 | 月度报告自动生成，TOP5/滞销/回款率分析 |

## 三、目标用户与场景

### 3.1 目标用户

| 角色 | 特征 | 核心诉求 |
|------|------|---------|
| 个体经销商 | 1-3人小团队，同时对接多个上游和客户 | 操作简单、数据安全、不依赖网络 |
| 小型装机店 | 有实体店面，兼顾零售和批发 | 库存精准、报价快速、对账清晰 |
| 区域代理 | 月成交量较大，需要经营分析 | 数据报表、趋势分析、滞销预警 |

### 3.2 核心使用场景

**场景 1：每周导入上游价格表**

上游发来新的 Word 价格表，用户一键导入，系统自动解析机型配置并保存快照。第二次导入时自动对比，提示新增/下架机型。

**场景 2：客户询价快速报价**

客户微信询价，用户在系统中选择机型，系统自动推荐报价区间，一键生成报价文本或图片发给客户。

**场景 3：出库发货**

客户确认购买后，使用条码枪扫描 SN 码，系统自动校验格式和重复，确认出库后扣减库存并生成出库确认单。

**场景 4：月底对账复盘**

月初点击"月度报告"，系统自动汇总上月销售额、毛利、回款率、TOP5 机型、滞销库存，辅助制定本月进货策略。

## 五、核心功能详细设计

### 5.1 机型管理

机型是系统的基础数据单元，一台机器由以下字段描述：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| 系列 | TEXT | 机型系列名，必填 | Y7000P、小新Pro16 |
| CPU | TEXT | 处理器型号 | i7-13700H、R7-7840H |
| 内存 | TEXT | 内存容量 | 16G、32G |
| 硬盘 | TEXT | 硬盘容量 | 512G、1T |
| 显卡 | TEXT | 显卡型号 | 4060、集成 |
| 屏幕 | TEXT | 屏幕尺寸 | 16、14 |
| 备注 | TEXT | 颜色、新品标识等 | 碳晶灰、新品 |

**Word 价格表导入**

系统支持从上游发来的 Word 价格表（.docx）自动解析机型信息。解析逻辑包括：

- 按已知机型前缀（Y7000P、小新Pro16、拯救者等）分割文本
- 正则提取 CPU、内存、硬盘、显卡、屏幕尺寸
- 兼容微软 Office 和 WPS 生成的 docx 文件
- 导入前提供预览，用户可勾选确认

### 5.2 库存批次管理

同一机型可能有多个进货批次，每个批次独立管理：

| 字段 | 说明 |
|------|------|
| 购入价 | 该批次的进货单价 |
| 进货数量 | 该批次初始进货台数 |
| 剩余数量 | 当前剩余库存（实时扣减） |
| 入库日期 | YYYY-MM-DD 格式 |
| 上游供应商 | 关联 suppliers 表 |
| SN 列表 | 该批次所有机器的序列号，逗号分隔 |
| 备注 | 含税/未税等说明 |

报价时自动扣减所选批次的剩余数量，库存不足时禁止报价并弹出提示。

### 5.3 报价记录与状态流转

报价记录是系统的核心业务流程载体，状态流转如下：

```
[创建报价] → 待确认 → [确认报价] → 已报价 → [出库操作] → 已出库 → [收款足额] → 已收款
                                          ↓
                                    [取消订单] → 已取消
```

各状态说明：

| 状态 | 说明 | 允许操作 |
|------|------|---------|
| 待确认 | 刚创建的报价，尚未正式向客户报价 | 确认报价、编辑、删除 |
| 已报价 | 已正式向客户报价 | 出库、收款、取消 |
| 已出库 | 货物已发出，库存已扣减 | 收款、取消 |
| 已收款 | 货款已全部收齐 | 仅查看 |
| 已取消 | 订单已取消 | 仅查看 |

### 5.4 收款与账款管理

系统支持部分收款，收款记录独立存储。当某笔报价的累计收款金额达到报价总额（报价单价 × 数量）时，系统自动将该报价状态更新为"已收款"。

支付方式包括：微信、支付宝、转账、现金。

客户对账单功能支持按客户 + 日期范围生成明细，导出 Excel。

### 5.5 数据备份与恢复

- **自动备份**：程序启动时自动复制数据库到 data/backup/ 目录，保留最近 7 个版本
- **JSON 导出**：全量导出所有表数据为 JSON 格式（含外键关系映射），保存到桌面
- **JSON 导入**：从 JSON 备份恢复数据，自动重建表关系
- **Excel 导出**：报价记录导出为 Excel，含状态颜色标记

## 六、智能技能（Skills）详细设计

### 6.1 报价决策助手

基于历史报价数据，为新报价提供决策参考。

| 输入 | 输出 |
|------|------|
| 机型特征（系列、CPU、内存、硬盘、显卡）<br>本次进货价（可选）<br>客户名称（可选） | 建议报价区间（最低/最高/中间价）<br>中间价对应的利润率<br>置信度（high/medium/low）<br>建议依据说明<br>历史报价统计（条数、区间、均价）<br>该客户历史成交价 |

算法逻辑：

1. 查询该机型的历史报价记录（状态为已报价/已出库/已收款）
2. 计算历史最低价、最高价、均价
3. 建议范围 = 历史均价 ± 5%
4. 如有该客户历史成交价，机型历史和客户历史各占 50% 权重
5. 无历史数据时，默认加价 3-10%

### 6.2 价格异动哨兵

每次导入 Word 价格表时自动保存快照，与上一次快照对比。

| 检测项 | 说明 |
|--------|------|
| 新增机型 | 本次出现、上次未出现的机型配置 |
| 下架机型 | 上次出现、本次未出现的机型配置 |

对比维度：系列 + CPU + 内存 + 硬盘 + 显卡 + 备注的规范化组合。

第一次导入时只保存快照，不产生报告；从第二次导入开始生成异动报告。

### 6.3 智能跟单提醒

扫描所有报价记录，找出超期（默认 3 天）未推进的订单：

- 待确认超过 3 天未转为已报价
- 已报价超过 3 天未出库
- 已出库超过 3 天未收款

按类别分组展示，包含客户名、机型、报价金额、超期天数。

### 6.4 月度经营报告

自动汇总指定月份的所有成交订单，生成经营分析：

| 指标 | 计算方式 |
|------|---------|
| 成交订单数 | 状态为已报价/已出库/已收款的订单数量 |
| 总销售额 | Σ(报价单价 × 数量) |
| 总毛利 | Σ((报价单价 - 购入价) × 数量) |
| 毛利率 | 总毛利 / 总销售额 × 100% |
| 回款率 | 已收款金额 / 总销售额 × 100% |
| 环比 | 与上月销售额/毛利对比变化率 |
| TOP5 机型 | 按毛利降序排列的前 5 个机型配置 |
| 滞销预警 | 有库存但本月无销售的机型，按占用资金降序 |

### 6.5 出库一条龙

出库流程的增强功能：

- **SN 批量扫描**：条码枪连续扫码，每扫一条自动换行；支持换行/逗号/空格分隔
- **自动去重**：重复的 SN 自动过滤
- **数量校验**：SN 数量与出库数量不一致时提醒
- **格式校验**：长度 6-30 位，只允许字母/数字/短横线
- **出库确认单**：包含客户名称、机型配置、单价、数量、SN 列表、检查项、签字栏

### 6.6 远程诊断助手

结构化故障排查决策树，支持以下故障类型：

| 故障类型 | 关键词 | 排查分支 |
|----------|--------|---------|
| 蓝屏 | 蓝屏、BSOD | 错误代码识别 → 安全模式引导 → 驱动/硬件分流 |
| 开不了机 | 黑屏、不开机 | 电源/屏幕/引导/重启四大分支 |
| WiFi 问题 | 断网、连不上 | 连不上/断连/网速慢三条路径 |
| 风扇噪音 | 风扇响 | 持续响/负载响/开机响三种场景 |

排查完成后可生成诊断报告，自动复制到剪贴板。
