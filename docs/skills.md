## Skill 1: 报价决策助手

**解决什么问题：** 每次报价都要翻记录、心算加价、纠结报多少钱。这个 Skill 基于你的历史报价数据，自动推荐一个合理的报价金额。

**工作原理：**

1. 你在左侧机型列表点击一台机器
2. 系统自动查询该机型的所有历史报价记录
3. 计算历史均价、最高价、最低价
4. 如果有该客户的历史成交价，还会综合参考
5. 自动填入建议报价金额到报价输入框

**你能看到什么：**

- 报价输入框自动填入建议金额（你可以直接用，也可以手动改）
- 选中机型后，报价依据：基于 N 条历史报价，历史区间 ¥XXXX-¥XXXX

**使用场景：**

- 新客户询价，不确定报多少合适
- 老客户复购，想参考上次成交价
- 市场波动时，避免报高了丢客户或报低了亏钱

**数据积累：** 用得越久，历史报价越多，推荐越准。建议坚持把每一笔报价都录入系统。

## Skill 2: 价格异动哨兵

**解决什么问题：** 上游每周发新价格表，你肉眼对比两份 Word 根本看不出哪些涨了、哪些降了、哪些新增了、哪些下架了。漏掉一个变动可能直接亏钱。

**工作原理：**

1. 每次你导入 Word 价格表时，系统自动保存一份「快照」
2. 和上一次快照做对比
3. 找出新增机型、下架机型
4. 弹出异动报告面板

**你能看到什么：**

导入价格表后自动弹出报告：

```
📊 价格异动报告（2026-05-14 → 2026-05-21）

[🔵 新增 (3)]  [⚠️ 下架 (1)]

--- 新增 Tab ---
小新Pro16GT  ultra9  32G  1T  ← 上游新上的配置

--- 下架 Tab ---
雷神E1  i5  16G  512G  ← 上游不再供货
⚠️ 请检查是否有库存需要尽快出货
```

**使用场景：**

- 每周导入新价格表后，立刻知道哪些变了
- 新增机型可以主动推给客户
- 下架机型赶紧检查库存，能出就出

**注意事项：**

- 第一次导入时只保存快照，没有对比基准
- 从第二次导入开始才会出报告
- 当前版本检测新增/下架，价格变动检测需要后续增强

## Skill 3: 智能跟单提醒

**解决什么问题：** 你报了一堆价，有些客户没回复你就忘了跟进。时间一长，客户找到别家了，订单就丢了。

**工作原理：**

1. 系统扫描所有报价记录
2. 找出「已报价超过 3 天未出库/收款」的订单
3. 找出「待确认超过 3 天」的订单
4. 找出「已出库超过 3 天未收款」的订单

**你能看到什么：**

点击工具栏 **🔔 跟单提醒** 按钮：

```
共有 5 条订单需要跟进：

📋 待确认（超过 3 天）：2 条
  • 张三 | 小新Pro16 | ¥4100 | 5 天前
  • 李四 | Y9000P | ¥7800 | 4 天前

📦 已报价未出库（超过 3 天）：2 条
  • 王五 | ThinkBook | ¥3500 | 6 天前

💰 已出库未收款（超过 3 天）：1 条
  • 赵六 | 拯救者 | ¥8200 | 7 天前
```

**使用场景：**

- 每天早上打开软件点一下，看看今天要跟进谁
- 月底催款前，看看哪些出库了但还没收到钱
- 避免「报了价忘了跟进」导致丢单

## Skill 4: 月度经营报告

**解决什么问题：** 月底对账只知道「这个月赚了多少钱」，但不知道哪个机型最赚钱、哪个是滞销品、回款率怎么样。

**工作原理：**

自动汇总本月所有报价记录，生成经营分析摘要。

**你能看到什么：**

点击工具栏 **📊 月度报告** 按钮：

```
📊 2026年5月 经营摘要
==============================

📦 成交订单: 42 单
💰 总销售额: ¥168,000
📈 总毛利: ¥9,800
📊 毛利率: 5.8%
💵 已收款: ¥131,000
📋 回款率: 78.0%

📉 环比: 销售额 ↑12.3%  |  毛利 ↑8.5%

🏆 最赚钱机型 TOP5:
  1. 小新Pro16 ultra5 | 卖12台 | 毛利 ¥3,600
  2. Y9000P 13900HX | 卖5台 | 毛利 ¥2,500
  ...

⚠️ 滞销库存预警:
  • 雷神E1 i5 16G 512G | 库存18台 | 占资金 ¥49,500 | 最早入库 2026-04-10
  • ThinkBook 14 i5 | 库存8台 | 占资金 ¥24,000 | 最早入库 2026-04-15

建议: 考虑降价促销或联系上游调货
```

**使用场景：**

- 每月初查看上月经营情况
- 发现滞销品及时清理
- 了解哪些机型值得多进货
- 环比数据帮你判断生意趋势

## Skill 5: 出库一条龙

**解决什么问题：** 出库时要录 SN、校验格式、检查重复、开出库单，步骤多容易出错。这个 Skill 把出库流程串成一条线。

**核心功能：**

### 5.1 SN 批量扫描

出库对话框的 SN 输入框已升级为多行文本框，支持：

- **条码枪连续扫入**：每扫一条自动换行，不需要手动按回车
- **多种分隔方式**：换行、逗号、空格都支持
- **自动去重**：扫重复的 SN 会自动过滤
- **数量校验**：SN 数量和出库数量不一致时提醒

### 5.2 SN 格式校验

自动检查每条 SN 的合法性：

- 长度在 6-30 位之间
- 只允许字母、数字、短横线
- 无效 SN 会标红提示

### 5.3 出库确认单

出库后可生成格式化的确认单文本，包含：

- 客户名称、机型配置、单价、数量
- SN 序列号列表（带勾选框）
- 外观/开机/配件检查项
- 客户签字栏

**使用场景：**

- 条码枪扫一下 SN 就录进去，不用手打
- 出库前自动校验，防止录入错误
- 打印确认单让客户签字，防碰瓷

## Skill 6: 远程诊断助手

**解决什么问题：** 客户打电话说「电脑蓝屏了」「开不了机了」，你需要在电话里引导他排查。靠经验记不住所有步骤，这个 Skill 把排查流程结构化。

**支持的故障类型：**

| 故障 | 关键词 |
|------|--------|
| 蓝屏 | 蓝屏、BSOD、蓝屏代码 |
| 开不了机 | 开不了机、黑屏、不开机、没反应 |
| WiFi 问题 | WiFi、断网、连不上网、网速慢 |
| 风扇噪音 | 风扇响、噪音大、散热 |

**工作原理：**

1. 点工具栏 **🔧 远程诊断** 按钮
2. 输入症状关键词，或直接点击快捷按钮
3. 系统显示排查流程，每一步有明确的问题
4. 根据客户的回答，点击对应选项
5. 系统给出处理方案，或进入下一步排查
6. 排查完成后生成诊断报告（自动复制到剪贴板）

**你能看到什么：**

```
🔧 蓝屏（Blue Screen）排查

❓ 蓝屏上有没有显示错误代码？

→ 有错误代码
→ 没有/看不清

[选择后显示处理方案]

📋 处理方案:
• IRQL_NOT_LESS_OR_EQUAL → 驱动冲突，进安全模式卸载最近安装的驱动
• KERNEL_SECURITY_CHECK_FAILURE → 系统文件损坏，运行 sfc /scannow
...

[📄 生成诊断报告]  ← 点击后自动复制到剪贴板
```

**使用场景：**

- 客户来电报修，直接打开诊断助手引导排查
- 减少无效送修，60% 的软件问题远程就能解决
- 生成的诊断报告可以发给客户留存
- 逐步积累排查经验，越用越顺手

**当前已内置的排查流程：**

- 蓝屏：含错误代码识别、安全模式引导、驱动/硬件分流
- 开不了机：电源/屏幕/引导/重启四大分支
- WiFi 断连：连不上/断连/网速慢三条路径
- 风扇噪音：持续响/负载响/开机响三种场景

后续可以在 `src/utils/remote_diagnose.py` 的 `DIAGNOSE_TREE` 字典中继续扩充更多故障类型。

## 技术架构

```
src/utils/
├── price_diff.py        # Skill 2: 快照存储 + 对比引擎
├── follow_up.py         # Skill 3: 超期订单扫描
├── monthly_report.py    # Skill 4: 月度数据聚合
├── shipment_flow.py     # Skill 5: SN 解析/校验/确认单
├── quote_assist.py      # Skill 1: 历史报价分析 + 推荐

```

所有 Skill 模块都是独立的，不互相依赖，可以单独使用也可以组合使用。
