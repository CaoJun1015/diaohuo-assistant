# IT 远程诊断助手

## Description

这是一个专为 IT 硬件经销商设计的远程故障诊断 Skill。当客户打电话报告电脑问题时，你不需要凭记忆去排查——这个 Skill 内置了结构化的故障排查决策树，能引导你一步步定位问题，给出处理方案，并生成可存档的诊断报告。

## When to Use

- 客户来电说电脑出问题了，你需要远程引导排查
- 客户描述的症状包括但不限于：蓝屏、黑屏、开不了机、WiFi断连、风扇噪音大、系统卡顿、无法充电等
- 你需要记录排查过程作为售后凭证
- 你想快速判断问题是软件还是硬件，决定是否需要送修

## Instructions

### 第一步：收集症状

向客户（或向你转述问题的人）确认以下信息：

1. **故障现象**：具体是什么问题？（蓝屏/开不了机/WiFi/风扇/其他）
2. **出现时机**：什么时候开始的？开机就出现还是使用中突然出现？
3. **复现条件**：做什么操作时会出现？是否每次都会出现？
4. **近期变更**：最近有没有装新软件、更新系统、摔过或进水？

### 第二步：匹配排查流程

根据收集到的症状，从知识库中选择对应的排查流程。本 Skill 内置了以下故障类型的排查决策树：

| 故障类型 | 触发关键词 |
|---------|-----------|
| 蓝屏 | 蓝屏、BSOD、蓝屏代码、IRQL、KERNEL |
| 开不了机 | 开不了机、黑屏、不开机、没反应、卡Logo |
| WiFi断连 | WiFi、断网、连不上网、网速慢、无线 |
| 风扇噪音 | 风扇响、噪音大、散热、温度高 |
| 系统卡顿 | 卡、慢、卡顿、响应慢 |
| 无法充电 | 不充电、充电慢、电池、电源 |

### 第三步：逐步引导排查

按照决策树的结构，**一次只问一个问题**，根据客户的回答选择分支：

1. 先问最能区分问题方向的问题（比如"蓝屏上有没有错误代码"）
2. 根据回答选择下一步
3. 每一步给出明确的操作指引（比如"按 F8 进安全模式"）
4. 如果客户不会操作，给出更详细的步骤说明

**重要原则：**
- 一次只给一个操作指令，不要一次性甩一堆步骤
- 用客户能听懂的语言，避免术语（除非客户本身懂技术）
- 如果客户说"不会操作"，换个更简单的方式引导

### 第四步：给出处理方案

排查到最终结论后，给出明确的处理方案：

- **软件问题**：给出具体修复步骤（卸载驱动、运行命令、系统还原等）
- **硬件问题**：建议送修，说明可能的硬件故障原因
- **不确定**：列出已排除的可能，建议下一步检测方向

### 第五步：生成诊断报告

排查结束后，自动生成一份诊断报告，格式如下：

```
============================
      远程诊断报告
============================
日期: YYYY-MM-DD HH:MM
客户: [客户名称]
机型: [品牌+型号]（如有SN码则记录）
SN码: [序列号]（如有）

故障现象: [一句话描述]

排查路径:
  1. [问题] → [客户回答]
  2. [问题] → [客户回答]
  3. ...

诊断结论: [软件问题/硬件问题/待进一步检测]

处理方案:
  [具体操作步骤]

后续跟进:
  [ ] 已解决
  [ ] 需要送修
  [ ] 需要回访确认

备注: [其他信息]
============================
```

将报告输出为文本，方便存档或发送给客户。

## Knowledge Base

### 蓝屏排查决策树

**第一步：蓝屏上有没有错误代码？**

- **有错误代码** → 根据代码查表：
  - `IRQL_NOT_LESS_OR_EQUAL` → 驱动冲突，进安全模式卸载最近安装的驱动
  - `KERNEL_SECURITY_CHECK_FAILURE` → 系统文件损坏，运行 `sfc /scannow`
  - `CRITICAL_PROCESS_DIED` → 关键进程崩溃，尝试系统还原
  - `PAGE_FAULT_IN_NONPAGED_AREA` → 内存问题，检查内存条是否松动
  - `VIDEO_TDR_FAILURE` → 显卡驱动问题，更新或回滚显卡驱动
  - `SYSTEM_SERVICE_EXCEPTION` → 系统服务异常，运行 `DISM /Online /Cleanup-Image /RestoreHealth`
  - `DPC_WATCHDOG_VIOLATION` → 驱动或硬件兼容性问题，更新所有驱动
- **没有/看不清** → 进入第二步

**第二步：蓝屏在什么情况下出现？**

- **开机就蓝屏** → 系统引导文件损坏
  - 重启，反复按 F8（Win10/11：按住 Shift 点重启 → 疑难解答 → 高级选项 → 启动设置）
  - 能进安全模式 → 运行 `sfc /scannow`，或使用系统还原点
  - 安全模式也蓝屏 → 硬件问题概率大，建议送修
- **使用中突然蓝屏** → 驱动或软件冲突
  - 进安全模式，卸载最近安装的软件/驱动
  - 运行 Windows 更新
  - 检查事件查看器中的错误日志
- **玩游戏/高负载时蓝屏** → 显卡驱动或散热问题
  - 更新显卡驱动到最新版本
  - 检查散热：清理风扇灰尘，确保出风口畅通
  - 用鲁大师/AIDA64 监控温度，CPU/GPU 超过 90°C 需处理散热
- **更新系统后蓝屏** → 系统更新问题
  - 进安全模式 → 控制面板 → 程序 → 查看已安装的更新 → 卸载最近的更新
  - 或使用系统还原点回滚

### 开不了机排查决策树

**第一步：按电源键后有什么反应？**

- **完全没反应（指示灯不亮、风扇不转）** → 电源问题
  1. 检查电源适配器是否插好，换一个插座试试
  2. 笔记本：长按电源键 15 秒放电，再重新开机
  3. 检查电源适配器指示灯是否亮
  4. 以上都不行 → 电源适配器或主板问题，建议送修
- **指示灯亮但屏幕不亮** → 屏幕或显卡问题
  1. 外接显示器试试（笔记本用 HDMI/Type-C 接口）
  2. 外接有画面 → 笔记本屏幕或排线问题
  3. 外接也没画面 → 显卡或内存问题
  4. 尝试拔电源 + 长按电源键 15 秒 + 重新开机
- **卡在品牌 Logo 界面** → 系统引导问题
  1. 等待 5 分钟看是否能自行进入系统
  2. 强制关机再开机
  3. 按 F2/Del 进 BIOS，检查硬盘是否被识别
  4. BIOS 能看到硬盘 → 系统引导损坏，需要 PE 修复或重装系统
  5. BIOS 看不到硬盘 → 硬盘可能坏了
- **反复重启** → 系统或硬件不稳定
  1. 开机按 F8 进安全模式
  2. 安全模式不重启 → 驱动或软件问题
  3. 安全模式也重启 → 硬件问题（内存/硬盘），建议送修

### WiFi 断连排查决策树

**第一步：WiFi 是连不上还是经常断？**

- **连不上 WiFi**
  1. 确认 WiFi 开关是否打开（Fn+F5 或侧边物理开关）
  2. 重启路由器和电脑
  3. 设备管理器 → 网络适配器，看无线网卡是否正常
  4. 无线网卡有黄色感叹号 → 右键卸载驱动，重启自动重装
  5. 设备管理器里看不到无线网卡 → 硬件故障
- **连上了但经常断**
  1. 更新无线网卡驱动（去品牌官网下载）
  2. WiFi 设置 → 电源管理 → 取消「允许计算机关闭此设备以节省电源」
  3. 尝试切换 2.4G/5G 频段
  4. 只在家里断 → 路由器问题；哪里都断 → 网卡硬件问题
- **网速很慢**
  1. 测速（speedtest.cn），对比其他设备
  2. 只有这台慢 → 更新网卡驱动
  3. 所有设备都慢 → 路由器或宽带问题
  4. 连 5G WiFi 测试，2.4G 容易受干扰

### 风扇噪音大排查决策树

**第一步：风扇是一直响还是特定情况才响？**

- **一直很响**
  1. 任务管理器检查 CPU 占用率，是否有程序占用过高
  2. 清理风扇灰尘（用吹风机冷风吹出风口）
  3. 检查散热模式：品牌自带软件里切换到安静模式
  4. 新机器就很响 → 可能风扇硬件问题
- **玩游戏/高负载时响** → 正常现象
  - 高负载风扇加速是正常的
  - 觉得太响：切换安静模式 / 降低游戏画质 / 用散热底座
- **开机没多久就响**
  1. 任务管理器看 CPU 占用率
  2. 结束不需要的后台程序
  3. 检查是否有 Windows 更新在后台运行

### 系统卡顿排查决策树

**第一步：是整体卡还是某个程序卡？**

- **整体都卡**
  1. 任务管理器看 CPU/内存/磁盘占用
  2. 磁盘 100% → 关闭 Windows Search 服务或 Superfetch 服务
  3. 内存不足 → 关闭多余程序，考虑加内存
  4. 机械硬盘 → 建议升级 SSD，提升最明显
- **某个程序卡**
  1. 重新安装该程序
  2. 检查是否是最新版本
  3. 兼容性问题 → 右键属性 → 兼容性模式运行

### 无法充电排查决策树

**第一步：电源适配器指示灯亮吗？**

- **不亮** → 适配器问题
  1. 换一个同规格适配器测试
  2. 检查充电接口是否有异物或松动
- **亮但不充电**
  1. 长按电源键 15 秒放电，重新插电
  2. 检查电池健康度（品牌自带软件或 `powercfg /batteryreport`）
  3. 电池膨胀 → 立即停止使用，送修更换
  4. 充电到一定百分比就停 → 可能开启了电池保护模式（联想养护模式等）

## Examples

### 示例 1：客户报告蓝屏

**输入：** 客户打电话说电脑蓝屏了

**Skill 执行流程：**

> 好的，我来帮你远程排查。先问客户一个问题：蓝屏画面上有没有显示错误代码？比如一串英文或者类似 `IRQL_NOT_LESS_OR_EQUAL` 这样的？

客户回答：有，写的是 `KERNEL_SECURITY_CHECK_FAILURE`

> 这个错误代码通常表示系统文件损坏。请引导客户执行以下操作：
>
> 1. 重启电脑
> 2. 在开机时反复按 F8（Win10/11：按住 Shift 点重启 → 疑难解答 → 高级选项 → 命令提示符）
> 3. 在命令提示符中输入：`sfc /scannow`
> 4. 等待扫描完成（约 10-15 分钟）
> 5. 如果提示修复成功，重启电脑
>
> 如果 sfc 无法修复，再执行：`DISM /Online /Cleanup-Image /RestoreHealth`

### 示例 2：客户报告开不了机

**输入：** 客户说按电源键完全没反应

**Skill 执行流程：**

> 请确认以下几点：
> 1. 电源适配器是否插好？指示灯亮不亮？
> 2. 换一个插座试试
> 3. 如果是笔记本，长按电源键 15 秒，然后松开，再按一次电源键

客户回答：适配器灯亮，长按后能开机了

> 好的，这是静电积累导致的，已经解决了。建议客户以后关机后不要一直插着电源，偶尔拔掉放放电。
>
> 生成诊断报告：静电积累导致无法开机，长按电源键 15 秒放电后恢复正常。
